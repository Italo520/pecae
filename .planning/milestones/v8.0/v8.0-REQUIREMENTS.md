# Requisitos da Milestone 8 (v8.0)

## Requisitos Funcionais

### M10 - Gestão de Assinaturas (DEFERRED)
- **RF110:** Sistema deve exibir tela de planos com preços, features e botão de upgrade (P1) - [DEFERRED]
- **RF111:** Sistema deve criar checkout Mercado Pago e retornar URL para WebView (P1) - [DEFERRED]
- **RF112:** Sistema deve processar webhooks do Mercado Pago e atualizar status de assinatura (P1) - [DEFERRED]
- **RF113:** Sistema deve bloquear cadastro de novos anúncios quando quota do plano atingida (M05) (P0) - [DEFERRED]
- **RF114:** Vendedor deve poder cancelar assinatura e downgrade para GRATUITO (P1) - [DEFERRED]

### M12 - Analytics e Dashboard
- [x] **RF120:** Vendedor deve ver dashboard com views, chats iniciados e rating médio dos seus anúncios (P2)
- [x] **RF121:** Vendedor deve ver gráfico de evolução de views dos últimos 30 dias por anúncio (P2)
- [x] **RF122:** Admin deve ver dashboard com DAU, novos cadastros, volume de anúncios e receita de assinaturas (P2)
- [x] **RF123:** Métricas devem ser atualizadas a cada 6 horas via BullMQ cron (não real-time) (P2)

## Regras de Negócio

### M10 - Gestão de Assinaturas (DEFERRED)
- **RN-M10-01:** Plano GRATUITO: máximo 3 anúncios PUBLISHED simultâneos
- **RN-M10-02:** Plano PRO: máximo 15 anúncios PUBLISHED simultâneos
- **RN-M10-03:** Plano PREMIUM: máximo 50 anúncios PUBLISHED simultâneos
- **RN-M10-04:** PECAÊ nunca processa dados de cartão diretamente — checkout externo via Mercado Pago
- **RN-M10-05:** Webhook deve ser verificado via HMAC-SHA256 antes de processar

### M12 - Analytics e Dashboard
- [x] **RN-M12-01:** Métricas de vendedor são privadas — apenas o próprio vendedor pode ver
- [x] **RN-M12-02:** Métricas globais (admin) são restritas a ADMIN e MODERATOR (RBAC)
- [x] **RN-M12-03:** ListingView registra apenas 1 view por IP por anúncio a cada 24h (dedup básico)
- [x] **RN-M12-04:** Analytics calculados sobre dados anonimizados — sem identificação de compradores individuais nos logs
