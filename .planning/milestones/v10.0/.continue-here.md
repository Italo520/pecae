# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: Auth Security — Zustand persist exibia tokens. — Mitigation: Web usará HttpOnly cookies para refreshToken e memória para accessToken. Mobile pode usar persist com `partialize` ou SecureStore. Não reintroduza tokens vulneráveis no storage.
- [ ] CONSTRAINT: Marketplace UX — O web-frontend tem escopo rigoroso focado em usabilidade de marketplaces top tier (chips de filtros, contadores dinâmicos, breadcrumbs). — Mitigation: Sempre verifique se o design não está "genérico". Deve haver social proof, layouts bem estabelecidos, loaders não triviais (Skeleton), etc.

**Do not proceed until all boxes are checked.**
