## 🔴 Principais Descobertas (Prioridade P0)
1. **Divergência Crítica de API**: O backend atende em `/admin/` para rotas de compatibilidade, mas o pacote `shared` e o Mobile estão tentando usar `/moderation/` (que não existe no backend).
2. **WebSocket Ausente no Mobile**: O Web utiliza STOMP para chat em tempo real via `useStomp.ts`, enquanto o Mobile não possui essa integração.
3. **Falta de Conformidade LGPD no Web**: A funcionalidade de exclusão de conta está presente no Mobile, mas ausente no Web.

## 🚀 Próximos Passos (Para a Próxima Sessão)
1. Começar abordando o item **P0 (Prioridade 0)** da divergência de moderação. 
2. A solução mais fácil, rápida e retrocompatível é adicionar um alias `@RequestMapping({"/admin", "/moderation"})` no controller `ControladorAdminCompatibilidade.java` do backend, de forma análoga a como foi feito em `CatalogoController.java`.
3. Validar se essa alteração no backend resolve as requisições no Mobile/Web.
4. Planejar a inclusão do WebSocket STOMP no app React Native (Expo).
